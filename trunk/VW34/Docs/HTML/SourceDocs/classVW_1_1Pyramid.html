Detailed Description

template<class StorageType>
class VW::Pyramid< StorageType >

Construct an image pyramid from an image.

Author:
ID Reid, JGH Knight
Date:
02/08/2002 The image pyramid consists of subsampled octaves of the base image, but whereas the input image can be in any format (providing it can be converted to the storage type), the pyramid is always monochrome (perhaps should PyramidMono?). Subsampling can be performed either using the correct method of Gaussian smoothing at the nyquist frequency followed by decimation, or using the faster method of averaging 2x2 blocks of pixels before decimation.

Compute the pyramid based on the supplied image (note that the pyramid is monochrome regardless of the input image type).

Only call the compute function if you didn't construct the object with an image. If quick and dirty is false (default) then Ian's method is used, where convolution is with an approximately Gaussian kernel. The alternative method just averages 2x2 blocks of pixels, which may give some marginal speed improvements.
