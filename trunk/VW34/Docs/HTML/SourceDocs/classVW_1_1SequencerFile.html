Detailed Description

A sequencer for handling sequentailly ordered files.

Author:
ID Reid, BJ Tordoff
Date:
02/08/02 The file sequencer can take several ordered lists of files and present each list as a channel within the sequencer paradigm. The sequence is initialised using just the first frame, from which the numbering scheme is deduced. The enumeration of the files must be using a zero-filled fixed-width field such as "some_name_00001.ppm". Channels can be added in one go by passing a vector of filenames, or singly by repeatedly adding a single filename. The netpbm package is used to allow loading of a wide sleection of image types, of which jpeg,png,gif,ppm,pgm,eps are just a few.

void VW::SequencerBase::SetDelay ( int  d  )  [inline, virtual, inherited]

Set the delay between frames (only usually used for non-hardware sequencers).

This is the time (in milliseconds) for which the sequencer sleeps between frames: the total frame period will be this plus the time taken to acquire the new frame. By default the delay is set to 1ms (zero or less is not a valid setting).
