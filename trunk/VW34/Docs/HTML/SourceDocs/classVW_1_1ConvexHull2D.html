Detailed Description

Computation of the convex hull of a set of 2D points.

Author:
BJ Tordoff
Date:
03/02/2004 The input is always a set of VW::Point2D's, but the output can either be a list of indices into this array, or a new array containing the points on the hull.

Computation time is approximately n^2 + n*log(n) for best case, and 2*n^2 at worst. The convex hull computation itself is just n*log(n) (if I coded it right), but requires pre-sorted corner lists, which can take up to n^2 operations. If the points happen to be in y-index order (such as output from corner detection), the sort will be pretty quick.
