#define _USE_MATH_DEFINES

In the following there are a few places where "Fast" and "Proper" codes exist, with one set commented out The Fast code avoids dividing gradients by 2, and avoids taking sqrts.

The resulting edge strengths are 4*[conventional_edge_strength]^2 The ONLY place this makes a difference is that the position of the interpolated sub-pixel edge is changed. As that quadratic interpolation was always a dubious hack anyway, one can live with it! The orientations are unchanged, as they are formed from ratios of gradients. The speed up is around 10%, so worth having.

DWM
