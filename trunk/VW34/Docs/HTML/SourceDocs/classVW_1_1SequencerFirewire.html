Detailed Description

Retrieve a sequence of image from one or more firewire cameras.

A sequencer for handling firewire digital cameras (not DV!).

This class implements the sequencer API for firewire cameras, providing easy access to continuous or single-stepped image sequences. The camera DMA is started when either 'Continuous' or 'Next' is first called and is only stopped on object destruction (largely due to the fact it seg-faults when the DMA is stopped). Conversely the grabbing thread is stopped when 'Stop' is called, so there should be no CPU overhead to leaving DMA running.

Author:
ID Reid, BJ Tordoff
Date:
02/08/2002 The firewire sequencer uses the FirewireCamera class to provide a sequencer-style interface to any digital cameras connected. Each connected camera is given a channel, and at present it is assumed that you want to use them all (ie. there is no facility for starting and stopping particular cameras).

void VW::SequencerBase::SetDelay ( int  d  )  [inline, virtual, inherited]

Set the delay between frames (only usually used for non-hardware sequencers).

This is the time (in milliseconds) for which the sequencer sleeps between frames: the total frame period will be this plus the time taken to acquire the new frame. By default the delay is set to 1ms (zero or less is not a valid setting).
