Detailed Description

template<class PixelType, class PixelPtr>
class VW::ImageBase< PixelType, PixelPtr >

The base class from which all images are made.

Author:
ID Reid, BJ Tordoff
Date:
02/08/02 The base image class provides much of the functionality for the specific image classes, but more importantly as an abstract class enforces consistency between them. All images contain a RIO (region of interest) which specifies which region of the image is valid (many convolutions and similar leave a border of unprocessed pixels), and when designing algorithms one should work only inside the specified ROI.

Note that the '=' operator is deliberately disabled for all image classes to prevent nasty accidents. If you really need to copy an image, use one of the ImageCopy functions provided.

IO can be performed on a range of standard image formats using the ReadImage() and WriteImage() functions, or, in order to preserve all attributes with the Read/WriteBinary() functions. The latter pair use an internal format which is similar to the PPM raw format, ie type and size are specified in plain text and pixel data as binary. Note that when reading a raw VW image the format (RGB, YUV etc), type (Char, UChar, Double etc) and size must match exactly - no conversion will be performed.

PixelPtr* VW::ImageBase< PixelType, PixelPtr >::pixel

Pointers to the start of each row.

A pointer to the start of the row r is given by image.pixel[r], and a pixel at row r, column c, by image.pixel[r][c]. This is functionally identical to saying image[r][c], but in some circumstances may be faster.
