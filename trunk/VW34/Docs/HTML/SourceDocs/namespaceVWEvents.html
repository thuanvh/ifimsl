Detailed Description

Keith Connor <krc@robots.ox.ac.uk> 20/06/03

A cunning system for setting up connections to general functions. Supports functions and member functions.

e.g., you write a library that uses Events, then others can make your library call their class member functions -- Cool!

An idea stolen from libsigc, but is less general, simpler and fits into one header.

Semantics: functions are attached to Events.

Syntax: Provider of library: Event0<void> event; // to set up a Event event(); // to execute the event handler User of library: if(you_like_functions) event.Attach(&myfunction); else if(you_like_member_functions) event.Attach(&myclass,&MyClass::Member); else if(you_want_to_chain_events) event.Attach(&other_event); else if(youve_finished_with_the_Event) event.Detach();
