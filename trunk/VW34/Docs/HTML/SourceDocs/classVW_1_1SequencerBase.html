Detailed Description

The base class from which all sequencers are made.

Author:
ID Reid, BJ Tordoff
Date:
02/08/2002 All sequencers must provide a mechanism for moving on a single frame, continuously getting frames and stopping. A frame may consist of several channels, but the frame is not presented to the user until all channels have provided an image (ie. until the slowest device responds). When a new frame is ready the NewFrameEvent signal is emitted and the user can retrieve images using the CopyImage function that all sequencers provide.

For many reasons (mainly hardware related) it is usually only possible to instantiate a single sequencer. Nothing in this base class prevents it, but bad stuff will probably happen.

NB: You *must* call the Init() function of a sequencer before use.

void VW::SequencerBase::SetDelay ( int  d  )  [inline, virtual]

Set the delay between frames (only usually used for non-hardware sequencers).

This is the time (in milliseconds) for which the sequencer sleeps between frames: the total frame period will be this plus the time taken to acquire the new frame. By default the delay is set to 1ms (zero or less is not a valid setting).

VWEvents::Event1<void,int> VW::SequencerBase::NewFrameEvent

The "new frame has arrived" event.

Attach a function to this callback signal if you want to perform an operation each time a new image arrives. This is typically done using my_sequencer.NewFrameEvent.connect(&MyFunction); where 'MyFunction' must take an integer argument and return void.
