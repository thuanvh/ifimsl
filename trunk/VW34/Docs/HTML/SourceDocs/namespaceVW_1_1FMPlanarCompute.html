VNL::Vector<double> VW::FMPlanarCompute::GenerateParameterVectorHZJK ( const VW::FMatrixPlanar &  fmatrix  ) 

Parameterises F as fixed lines lh and ls, from the symmetric part of F, and two two-vector [a1 b1]' and [a2 b2]' which define the position of the epipoles e1 and e2 on lh (the horizon line).

So F = [e2]x [ls]x [e1]x as per HZ, but e1 = a1 xs + b1 xp, and e2 = a2 xs + b2 xp, where xs = lh x ls, and xp = xs x lh (ie xp is a point on lh perpendicular to xs, the intersection of lh and ls) We have from this that a1|2 = xs' e1|2, b1|2 = xp' e1|2.

VNL::Vector<double> VW::FMPlanarCompute::GenerateParameterVectorVLJK ( const VW::FMatrixPlanar &  fmatrix  ) 

Parameterises F as antisymmetric part and symmetric part, but symmetric part is parameterised as fixed lines lh and ls as per Vieville and Lingrand (ECCV96).

So F = [fa]x + (lh.ls' + ls.lh')

ERROR: It turns out this is an under-parameterisation, as fa is a point on lh and therefore only constitues one parameter, not two.
